# Experimental Mode

## Purpose
`experimental` mode exists to preserve and exercise the future-facing autonomous architecture without making it production-active for compliant legal flows.

## Rules
- must be explicitly selected
- autonomous execution must be explicitly enabled
- should be treated as non-operative for legal validity
- may be used for shadow benchmarking and architecture development

## What It Is Not
- not a shortcut around compliant mode
- not a silent fallback
- not a replacement for the human-supervised authority path in current legally operative operation
